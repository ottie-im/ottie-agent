# skill-approve — 审批流程

## 职责
管理消息从"Agent 拟稿"到"用户确认后发出"的完整审批流程。

## 流程
1. 收到 skill-rewrite 的改写结果
2. 创建 ApprovalRequest，推送给用户
3. 用户选择：批准 / 编辑 / 拒绝
4. 批准 → 发送消息
5. 编辑 → 用编辑后的内容发送
6. 拒绝 → 丢弃，不发送

## 规则
1. 每条消息都必须经过审批（早期阶段）
2. 审批超时（5分钟）自动丢弃
3. 保留审批历史，用于后续信任委托的训练数据

## 输入
- draft: 改写后的消息
- originalIntent: 用户原始意图
- targetRoom: 目标房间

## 输出
- ApprovalRequest 对象，等待用户决策
